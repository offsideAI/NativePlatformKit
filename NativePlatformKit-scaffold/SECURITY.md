# Security Policy

## Supported versions

NativePlatformKit is pre-1.0. Until the first stable release, security fixes are made on the latest
released version only.

| Version | Supported |
|---|---|
| latest `0.x` release | ✅ |
| older `0.x` releases | ❌ |
| `main` (SNAPSHOT) | ✅ (best effort) |

Once `1.0.0` ships, this table will list the supported major/minor lines.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues, discussions, or pull
requests.**

Report privately through either of:

1. **GitHub Security Advisories** — preferred. Go to the repository's **Security → Report a
   vulnerability** tab to open a private advisory.
2. **Email** — **chatoffside@gmail.com** with the subject line `SECURITY: NativePlatformKit`.

Please include, as far as you can:

- the affected version(s) and platform,
- a description of the vulnerability and its impact,
- step-by-step reproduction (a minimal sample or failing test is ideal),
- any known mitigations or workarounds.

## What to expect

- **Acknowledgement** within **3 business days**.
- An initial assessment and severity triage within **10 business days**.
- We will keep you informed of progress and coordinate a disclosure timeline with you. We aim to
  release a fix and a coordinated public advisory within **90 days** of triage, sooner for
  high-severity issues.
- With your consent, we will credit you in the advisory and release notes.

We follow coordinated disclosure and ask that you give us a reasonable opportunity to remediate
before any public disclosure. Thank you for helping keep NativePlatformKit and its downstream
consumers safe.
